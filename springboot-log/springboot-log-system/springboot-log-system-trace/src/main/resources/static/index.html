Spring Cloud Sleuth 自动追踪 + Zipkin 可视化 | traceId/spanId 自动注入日志

1. 检查库存（自动追踪 + 手动 Span）

2. 扣减库存

3. 跨服务调用（RestTemplate 自动传播 traceId）

RestTemplate 会自动在 HTTP Header 中注入 X-B3-TraceId，目标服务也集成 Sleuth 时可实现链路关联。

4. 查看当前链路信息

获取当前请求的 traceId 以及 Sleuth 自动追踪的说明信息。

架构说明

数据流向：Spring Boot + Sleuth → HTTP/Kafka → Zipkin Server → Zipkin UI

自动追踪：Sleuth 自动追踪 HTTP 请求、RestTemplate、异步方法、MQ 消息等

日志格式：[appName, traceId, spanId, exportable] 自动注入每条日志

传播机制：通过 HTTP Header（X-B3-TraceId / X-B3-SpanId）跨服务传播

Zipkin UI：http://localhost:9411 查看链路详情和服务依赖拓扑
